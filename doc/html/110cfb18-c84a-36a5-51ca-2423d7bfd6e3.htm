Classes
  ClassDescription
BufferedClient
BufferedClient class for the OTEX framework. Unlike the regular client, which requires you to manually calculate operations, the BufferedClient keeps a local cache and generates operations based on a diff algorithm. Less flexible than Client, easier to get going "in a pinch".

Client class for the OTEX framework.

Server class for the OTEX framework.
ServerStartParams
Bundle of parameters for Server.Start (so the parameter list isn't enormous).
ServerDescription
Packet of information describing a public OTEX server. Unlike other packets, this is not handled using the PacketStream, so does not implement IPacketPayload (these are sent over UDP instead).
ServerListener
Class responsible for listening for the availability of public OTEX servers.
ThreadController
Base class of OTEX framework objects which own threads.
ThreadControllerThreadException
General-purpose exception class for capturing other exceptions thrown on a thread managed by a ThreadController subclass.
